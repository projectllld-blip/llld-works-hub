# AGENTS.md — LLLD Works Hub

## まず読む
- グローバルの「LLLD制作方針」（CLAUDE.md / AGENTS.md）を必ず先に読むこと。
- 共通ルールはそちら。ここにはこのアプリ固有のことだけ書く。

## このアプリは何か
社内の各HTMLツール（だこくん / SeatFlow / PDFツール 等）を、
一覧・検索・お気に入り・利用履歴で束ねる「社内ツールポータル」。
目的は「毎日使うツールにすぐ戻れる」こと。

## 対象ユーザー
LLLD社内スタッフ。
（※将来は「クライアント向けツールボックス」「一般向けマーケット」へ拡張する構想あり。下の技術メモ参照）

## 壊してはいけないもの（このアプリ固有）
- コンテンツ登録・一覧表示
- カテゴリ / お気に入り / 利用履歴
- 各ソート（最終利用順 / 更新日順 / 名前順 / カテゴリ順）
- 登録データ（localStorage）と、そのエクスポート / インポート

## 固有の技術メモ
- ★ コンテンツ一覧の「データの読み込み元」は、関数1つ（例: loadContents()）に集約しておくこと。
  呼び出し側は変えずに、中身だけ localStorage → data/contents.json → 外部API と
  差し替えられる形を維持する。将来の拡張（社内共有 / クライアント配布 / 販売マーケット）が
  この1点に乗るので、ここを散らかさない。
- 各ツールへのリンクは相対パス基本。GitHub Pages で崩れないこと。

## GitHub 連動ルール

このリポジトリ内のアプリ（`apps/` 以下）に変更を加えたら、必ず GitHub へも反映すること。

手順:
1. 変更対象ファイルだけを `git add` でステージする（無関係の変更を混ぜない）。
2. 変更内容を端的に示すコミットメッセージで `git commit` する。
3. `git push origin main` で GitHub に反映する。
4. push できなかった場合は理由と未反映コミットの有無を完了報告で伝える。
