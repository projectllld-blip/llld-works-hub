# 08 PROMPT QUALITY RULES

目的:
ChatGPTが作っていた高品質なCodex指示の型をrepo内に残すこと。

## 品質ルール
- 作業目的を先に書く。
- 今回やらないことを明確にする。
- 触ってよいファイル / 触ってはいけないファイルを分ける。
- STOP条件を毎回入れる。
- 確認項目を具体的にする。
- 報告フォーマットを固定する。
- 次にCodexへ投げる指示を作らせる。
- 曖昧な仕様は勝手に決めず `HUMAN_REQUIRED: YES` にする。
- UIUX・Supabase・外部サービス・本番判断は人間確認に戻す。
- 古い会話ログではなく、repo内ドキュメントを正とする。

## よい指示に含めるもの
- Phaseまたは作業名。
- 目的。
- 現在地の要約。
- 変更対象。
- 変更禁止対象。
- STOP条件。
- 確認項目。
- 完了条件。
- 報告フォーマット。

## 避ける指示
- 「いい感じに直して」のような目的が曖昧な指示。
- 本体機能と運用整備を混ぜる指示。
- secret入力や外部サービス設定をCodexに任せる指示。
- UIUX最終判断をCodex単独に任せる指示。
- 触ってはいけないファイルを書かない指示。
