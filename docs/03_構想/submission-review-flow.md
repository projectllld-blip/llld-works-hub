# Submission Review Flow

外部投稿コンテンツを掲載する場合の審査制フロー。

## 基本方針

- 外部投稿者が直接公開できる仕組みはまだ作らない。
- まずはLLLDが内容を確認し、手動で掲載する。
- 投稿者情報、審査状況、権限情報はlocalStorageに保存しない。

## 流れ

1. 投稿相談を受ける。
2. コンテンツの種類を確認する。
3. 著作権チェックを行う。
4. 個人情報チェックを行う。
5. 有料販売する場合は納品方法を確認する。
6. LLLDが掲載可否を判断する。
7. `data/contents.json` に掲載情報を追加する。
8. ローカル検証版で表示確認する。
9. 本番反映前に `release-checklist.md` を通す。

## 投稿できる候補

- HTMLアプリ
- Excelテンプレート
- PDF資料
- Word資料
- チェックリスト
- マニュアル
- 授業資料
- 面談資料
- 補助金資料
- 業務改善テンプレート

## NG例

- 権利確認ができない教材PDF
- 入試問題の無断掲載
- 個人情報を含む資料
- 顧客や生徒の実データ
- 契約書や見積書の実ファイル
- APIキーや秘密情報を含むHTML/JS
