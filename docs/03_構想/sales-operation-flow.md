# Sales Operation Flow

LLLD Works Market の手動販売・手動納品フロー。

## 基本方針

- 最初は自動決済、自動納品をしない。
- 購入希望や開発相談を手動で確認する。
- 有料商品の本体ファイルはGitHub Pagesに置かない。

## 購入相談の流れ

1. ユーザーが商品詳細またはMarketカードから購入相談へ進む。
2. `request.html` で相談内容を作成する。
3. ユーザーがメール送信または内容コピーで連絡する。
4. LLLDが内容と対象商品を確認する。
5. 支払い方法を案内する。
6. 入金確認後、メール添付またはGoogle Drive限定リンクで納品する。

## 開発相談の流れ

1. ユーザーが開発相談へ進む。
2. 現在の困りごと、利用場面、希望納期を確認する。
3. 必要機能を整理する。
4. 見積もりを案内する。
5. 制作範囲が決まってから着手する。
