# エラー処理・空状態 実装指示

## 基本方針
- 画面が空白にならないようにする。
- エラーと空状態を分ける。
- ユーザーが次に取れる行動を短く示す。
- 開発者向けの詳細原因、SQL、RLS本文、token、secretは画面に出さない。

## 表示する状態
- 読み込み中
- 未ログイン
- データなし
- 読み込み失敗
- 保存失敗
- 再試行案内

## 文言の考え方
- 未ログインは「読み込み失敗」として扱わない。
- データなしはエラー扱いにしない。
- 保存失敗は成功扱いにしない。
- Supabase確認が必要な場合は、画面では分かりやすく案内し、詳細はdocsや人間確認に残す。
