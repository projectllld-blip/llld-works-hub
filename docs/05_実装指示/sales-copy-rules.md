# Sales Copy Rules

LLLD Works Market の商品説明文ルール。

## 誰に向けるか

- 小規模事業者
- 塾
- 教室長
- 店舗運営者
- 事務担当者
- 現場スタッフ

ITに詳しくない人が読んでも分かる言葉にする。

## 書き方

- 誰の何の困りごとを解決するかを先に書く。
- 「何ができるか」はその後に書く。
- 短く、具体的に書く。
- SaaS用語や開発者向けの言葉を使いすぎない。
- 日本の小さな現場で使う前提の言葉にする。

## 避ける表現

- 誇大表現
- 効果を保証する表現
- 大企業向けSaaSのような抽象語
- 技術用語だけの説明
- 未完成なのに完成品のように見える表現

## 状態の書き分け

- 未完成: 準備中
- 試用中: β版
- 手動対応: 相談受付
- 有料だが決済未接続: 購入について相談する
- 開発案件: 開発を相談する

## 有料商品の表現

- 有料商品の本体ファイルはGitHub Pagesで直接配布しない。
- 購入希望後、支払い方法と納品方法を手動で案内する。
- 納品方法はメール添付、Google Drive限定リンク、手動共有を前提に書く。

## NG例

- 絶対に売上が上がる
- 完全自動で解決
- 誰でも必ず成果が出る
- 即日納品保証
- 公式制度に完全対応

制度、補助金、教材、個人情報が関わるものは、確認責任と利用範囲を明記する。
