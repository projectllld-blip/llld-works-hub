# 表側販売表示ルール

## 目的

Marketの表側では、見た人がすぐに以下を判断できるようにする。

- 何の商品か
- 誰向けか
- 無料、β版、有料、相談、準備中のどれか
- 価格はいくらか
- どう使う、またはどう相談するか

## Marketカード

カードでは以下を優先して表示する。

- タイトル
- 短い説明
- 価格表示
- 販売状態
- 対象者
- 提供方法
- カテゴリ
- 投稿者
- 詳細ページへの導線
- 状態に応じたCTA

詰め込みすぎず、長文説明は詳細ページに回す。

## 詳細ページ

詳細ページでは以下の順に伝える。

1. 誰向けの商品か
2. 何に困っている人向けか
3. 何ができるか
4. 何が含まれるか
5. どう使うか
6. 注意点
7. 相談・購入CTA

空データは無理に表示しない。

## CTA文言

- `free`: 無料で使う
- `free-beta` / `beta`: β版を試す
- `paid` + `on-sale`: 購入する、または購入について相談する
- `paid` + `preparing`: 先行案内を受ける
- `consultation`: 開発を相談する
- `coming-soon`: 先行案内を受ける
- `internal`: 社内限定

Market側には `internal` / `internal_only` を表示しない。

## 文言トーン

- 日本の小規模事業者、塾、教室長、現場担当者に伝わる言葉にする。
- ITっぽすぎる表現やSaaS用語を増やしすぎない。
- 「何ができるか」より「何に困っている人向けか」を先に書く。
- 未完成のものは、β版、準備中、相談受付と明記する。
- 有料商品はGitHub上で直接配布しない前提で書く。
