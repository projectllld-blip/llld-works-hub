# GitHub Pages Risk

GitHub Pages運用で注意するリスク。

## 前提

GitHub Pagesは静的ホスティングです。置いたファイルは基本的に公開ファイルとして扱います。

## 主なリスク

- URLを知っている人がファイルを閲覧できる可能性がある。
- JSON、JS、画像、PDFなども直接アクセスされる可能性がある。
- フロントJSに書いたAPIキーや秘密情報は隠せない。
- 有料商品の実ファイルを置くと、購入前に取得される可能性がある。
- 個人情報や顧客情報を置くと情報漏えいになる。

## GitHub Pagesに置いてよいもの

- 公開LP
- 無料公開するHTMLアプリ
- 商品説明
- 公開可能なサムネイル
- 公開可能なサンプルデータ
- 問い合わせ導線

## GitHub Pagesに置かないもの

- 有料商品の本体ファイル
- 顧客資料
- 教材PDF
- 契約書、見積書、請求書
- 勤怠・名簿・面談記録などの実データ
- APIキーや秘密情報

## 運用ルール

- 有料商品は手動納品にする。
- 納品はメール添付、Google Drive限定リンク、手動共有を使う。
- 本番反映前に `release-checklist.md` を確認する。
- 外部投稿コンテンツは `market-content-review-checklist.md` を通してから掲載する。
