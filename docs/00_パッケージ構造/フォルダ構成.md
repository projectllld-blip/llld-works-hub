# フォルダ構成

## このパッケージごと渡せば動く範囲

以下は動作に必要な本体です。原則として、パスを変えるとリンクやJSON読み込みが壊れる可能性があります。

```text
/
├── index.html
├── marketplace.html
├── content-detail.html
├── author.html
├── request.html
├── thanks.html
├── submit.html
├── assets/
├── data/
├── apps/
├── contents/
├── favicon.ico
└── .nojekyll
```

## 仮DB

```text
data/
├── contents.json
├── authors.json
└── categories.json
```

今はJSONを仮DBとして使います。将来Supabase等へ移す場合も、画面側は `assets/js/contentService.js` を経由する前提にします。

## 資料

```text
docs/
├── 00_パッケージ構造/
├── 01_セットアップ/
├── 02_監査/
├── 03_構想/
├── 04_フェーズ記録/
├── 05_実装指示/
└── context/
```

Markdown資料は原則 `docs/` に置きます。ただし、`README.md` と `AGENTS.md` はパッケージの入口なのでルートに残します。

## アーカイブ

```text
archive/
```

元資料、受信箱由来の資料、後から参照する可能性がある素材を置きます。動作に必要なものはここに置かないでください。

ルートに残っていた古い単体HTMLは、現行導線から参照されていないため `archive/legacy-root-html/` に移動しています。
