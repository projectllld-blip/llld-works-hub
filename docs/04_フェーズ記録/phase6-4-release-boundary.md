# Phase 6.4 本番反映境界整理

作成日: 2026-06-25
対象: LLLD Works Hub / Works Market ローカル検証版 v0.6

## Phase 6.4 の目的

新機能を追加せず、v0.6 の未コミット差分を整理し、本番 GitHub Pages へ反映できる候補とローカル検証に留めるものを切り分ける。

このPhaseでは commit / push は行わない。

## 今回やったこと

- `git status --short` で未コミット差分を確認した。
- `git diff --name-only` で追跡済み変更ファイルを確認した。
- `git diff --check` で空白エラーがないことを確認した。
- 秘密キー、Supabase本番接続、Stripe系文字列の混入を検索した。
- zip、env、PDF、Office系ファイルの所在を確認した。
- `アーカイブ.zip` を本番反映禁止候補に分類した。
- `archive/受信箱由来_2026-06-24/コンテンツポータル構想.pdf` を本番反映禁止候補に分類した。
- `.gitignore` に zip / env 系の除外ルールを追加した。
- v0.6 のリリース境界を `docs/02_監査/release-boundary-v0.6.md` に記録した。

## まだ本番反映していないこと

- 本番 GitHub Pages への反映
- commit
- push
- Supabase 接続
- 認証
- 決済
- 自動納品
- 管理画面からの実データ更新
- 有料商品の本体ファイル配置

## 次に進む候補

- 本番反映対象だけを stage する前提のリリース候補リスト作成
- `admin.html` と `docs/` を本番公開対象から外す運用整理
- `アーカイブ.zip` をリポジトリ外へ移動
- 本番反映前の最終ブラウザ確認
- v0.6 リリース候補 commit の作成判断

## 残リスク

- `docs/` には内部方針が含まれるため、公開範囲の判断が必要。
- `archive/` 配下のPDFは本番公開対象にしない前提で扱う必要がある。
- `admin.html` はモックだが、管理画面に見えるため本番公開時は導線から外す必要がある。
- `data/contents.json` の有料商品は手動販売・手動納品前提であり、自動決済や購入履歴管理はまだない。
- 教材系・資料系コンテンツは、本番公開前に権利確認が必要。
