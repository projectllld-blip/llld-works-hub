# Phase 6 管理画面Skeleton記録

作成日: 2026-06-25

## 目的

将来の投稿管理、審査制、販売管理、アカウント制に備えて、管理画面の骨組みだけを用意する。

## 作成したもの

- `admin.html`
- `assets/js/adminMockPage.js`

## 現在できること

- `data/contents.json`、`authors.json`、`categories.json` を読む。
- コンテンツ件数、Market表示件数、販売中件数、審査・準備中件数を確認する。
- コンテンツ一覧を検索する。
- 公開状態で絞り込む。
- 表示確認用のモックロールを切り替える。
- Phase 6.1で、詳細、審査、編集モック、公開判定、投稿申請一覧モックを追加した。

## できないこと

- 保存
- 公開
- 非公開化
- 削除
- 権限変更
- 購入履歴管理
- 売上管理
- 自動納品
- Supabase接続

## 安全方針

- 本番操作不可のローカルモックとして扱う。
- localStorageに保存するのは表示確認用モックロールのみ。
- 本番権限、審査状況、販売履歴はlocalStorageへ保存しない。
- 本番化する場合は、Supabase等のDBとRLSを前提に再設計する。
