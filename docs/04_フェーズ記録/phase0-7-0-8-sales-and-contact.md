# v0.7 / v0.8 表側販売強化・問い合わせ接続記録

作成日: 2026-06-25

> 注記: この記録は初期の問い合わせ接続方針です。v0.8修正版以降は、Googleフォーム連携ではなく `request.html` の自社フォームUIを使います。現行ルールは `phase0-7-0-8-sales-and-owned-form.md` と `docs/05_実装指示/owned-contact-form-rules.md` を正とします。

## 目的

v0.6 GitHub Pages確認版の公開後、表側を「何の商品か分かる」「誰向けか分かる」「相談へ進める」状態に近づける。

同時に、問い合わせ・購入相談の送信先を `data/site-config.json` で管理できるようにする。

## v0.7 表側販売強化

- Market冒頭に、無料、β版、有料、相談の違いを短く表示した。
- 商品カードに、価格、販売状態、対象者、提供方法、カテゴリ、投稿者、CTAを表示する方針にした。
- 商品カードから詳細ページへ進める導線を残した。
- 商品詳細ページのファーストビューに、対象者、販売状態、価格、提供方法を表示した。
- 詳細ページでは、悩み、できること、提供内容、使い方、注意事項を自然に表示する。
- 空項目は無理に表示しない。

## v0.8 問い合わせ・購入相談接続

- `data/site-config.json` を追加した。
- 現行の修正版ではGoogleフォーム連携を使わない。
- `contact.mode` を `demo`、`mailto`、`endpoint` で切り替える。
- demo modeでは実送信せず、入力内容の確認と準備中表示だけを行う。
- mailto modeではメール作成、endpoint modeでは将来自社APIへのPOST設計にする。
- request.html は `item` と `slug` の両方を受け付ける。
- `item` があれば `item` を優先し、なければ `slug` を使う。

## thanks.html の扱い

`thanks.html` はローカル検証用・導線確認用の完了画面として扱う。

自社フォームが未接続またはdemo modeの場合、偽の送信完了として扱わない。

## まだ実装しないこと

- Supabase接続
- 認証
- 決済
- 自動納品
- 購入履歴管理
- 投稿者ログイン
- 管理画面本番化

## 禁止事項

- 有料商品の本体ファイルをGitHubに置かない。
- 個人情報、顧客情報、勤怠実データ、教材PDF、契約書、見積書を追加しない。
- `internal` / `internal_only` はMarket側に出さない。
- `archive/`、zip、env系ファイルを戻さない。
