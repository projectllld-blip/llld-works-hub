# phase1-4-productization-policy

## Phase

v1.4 実アプリの商品化方針整理

## 状態

方針整理。

v1.3で、購入者向けの旧「アプリ追加申請」は本線から外した。v1.4では、既存の実アプリやコンテンツを、購入ページや決済より前にどのような商品状態として見せるかを整理する。

今回はdocs整理のみで、`data/contents.json`、本体HTML / JS / CSS、Supabase migration、RLS、DB、GitHub Actionsは変更しない。

## 目的

v1.5 販売用UI/UX強化、v1.6 購入ページへ進む前に、以下を明確にする。

- 実アプリごとの商品状態。
- 商品カードに出すものと相談導線へ分離するもの。
- 無料 / 有料化予定 / β版 / 準備中 / 利用中のCTA。
- 決済未実装中の有料予定アプリの扱い。
- 任意サポートの見せ方。
- 将来、管理者画面で編集したい商品表示項目。

## 商品状態の分類

### 無料

ユーザーがすぐ試せる入口。

CTA:

- 未利用: `利用開始` または現行表示では `無料で使う`。
- 利用中: `開く`。

方針:

- 決済不要。
- 利用開始後は将来的に `app_instances` に追加し、`account.html` の利用中アプリへ反映する。
- v1.4では実装しない。

### 有料化予定

将来は有料商品として扱いたいが、決済未実装中は無料またはβと同じように試せる状態。

CTA:

- 決済未実装中: `利用開始` / `β版を試す` / `購入について相談する` のいずれか。
- 決済実装後: `購入する`。

方針:

- 価格が未確定なら `価格未定` または `有料化予定` と表示する。
- 決済実装までは、購入者に「支払い済み」と誤解させない。
- `app_instances` 追加、購入履歴、決済状態更新は後続Phaseで扱う。

### β版

試しながら改善するアプリ。

CTA:

- `β版を試す`。

方針:

- 未完成範囲やPARKED範囲を明記する。
- SeatFlowなど、表示中レイアウトの限定保存はできても完全クラウド同期が未完了のものは、β版または相談導線として扱う。

### 準備中

まだ利用開始させない商品。

CTA:

- `準備中`。
- 必要に応じて `先行案内を受ける`。

方針:

- `app_instances` 追加や利用開始はしない。
- 購入ページや詳細ページでは、できることよりも準備中であることを優先して表示する。

### 相談導線へ分離

そのまま汎用商品として売るより、初期設定、カスタマイズ、導入支援、開発相談が必要なもの。

CTA:

- `相談する`
- `開発を相談する`
- `教室用に相談する`

方針:

- 通常の商品カードで「購入する」とは見せない。
- 問い合わせ、導入相談、カスタマイズ相談、開発相談へ分離する。
- 相談後に個別見積もり・導入支援へつなげる。

### PARKED

今は商品化しない範囲。

対象:

- SeatFlow完全クラウド同期。
- SeatFlow複数レイアウト同期。
- SeatFlow名簿 / QR / NFC / メモのクラウド保存。
- SeatFlow全体状態の完全バックアップ・復元。
- バックアップ復元実装。
- クリエイター向けアプリ投稿 / 掲載申請。
- 決済 / 購入履歴。
- スタッフ個別ログイン、権限管理、複数店舗管理。

## アプリ別の注意

### SeatFlow

SeatFlowは、表示中レイアウトの限定的なクラウド保存・読込は確認済み。ただし以下はPARKED。

- 複数レイアウト同期。
- 名簿 / QR / NFC / メモのクラウド保存。
- 全体状態の完全同期。
- 履歴・復元・競合解決。

そのため、v1.4時点では「完成済み有料商品」ではなく、β版 + 教室用相談導線として扱う。

### だこくん

標準版はβ利用可能な入口として扱う。勤怠は実務影響が大きいため、本格利用前には初期設定、使い方説明、出力確認、現場運用確認をサポート候補にする。

### PDF編集ツール

無料入口として扱う。PDFファイル本体はサーバーに保存しない。著作権物や個人情報の管理は利用者側で行うことを表示する。

### 小テスト作成ツール

β版として扱う。教材データや印刷導線の整備状況を明記し、教材運用相談は任意サポートへ分離する。

## 任意サポート方針

購入者向けの「アプリ追加申請」は使わない。

購入者には、購入前後で必要に応じて以下を選べる導線を用意する。

- 初期設定サポート。
- 使い方説明。
- カスタマイズ相談。
- 導入代行。

サポートはアプリの利用開始条件にしない。アプリ本体の購入 / 利用開始と、サポート申込は別の導線にする。
