# Phase 4 圧縮記録

作成日: 2026-06-25

## 判断

Phase 4 の「社内・関係者β」は、現時点では従業員がいないため通常の社内βとして実施しない。

ただしスキップではなく、代表本人による疑似βチェックに圧縮する。

## 圧縮後の確認内容

- `python3 -m http.server 5500` でローカル起動する。
- トップページ、Market、詳細、相談、thanks、管理モックを確認する。
- 有料商品、無料商品、開発相談商品の役割が分かるか確認する。
- 既存Hubの導線、最終利用履歴8件、お気に入り、検索、カテゴリ絞り込みが壊れていないか確認する。
- GitHub Pagesに置いてはいけない情報が混ざっていないか確認する。
- 価格、販売状態、CTA、手動納品の案内に誤解がないか確認する。

## 疑似βで見る観点

- 代表本人が説明なしで主要導線をたどれるか。
- ビジコン相談相手や協力者に見せても目的が伝わるか。
- 現時点で外部公開すると危険な情報がないか。
- UIの細部より、販売検証と安全運用に進める状態か。

## 次に通常βへ戻す条件

- 協力者、関係者、または社内メンバーが確認できる状態になる。
- 問い合わせ先、納品方法、販売商品の実体管理場所が確定する。
- 本番GitHub Pagesへの限定反映方針が決まる。

## 注意

疑似βチェックは、本番公開の代替ではない。
本番反映前には `docs/02_監査/release-checklist.md` を必ず通す。
