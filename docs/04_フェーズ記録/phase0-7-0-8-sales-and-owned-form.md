# v0.7 表側販売強化 + v0.8 自社フォーム化

作成日: 2026-06-25

## 目的

v0.7では表側の販売ページを、商品棚として見やすくする。
v0.8では問い合わせ・購入相談導線をGoogleフォームではなく、自社フォームUIとして整備する。

## v0.7 表側販売強化

- Marketページに、無料、β版、有料、相談の違いを表示した。
- 商品カードで、タイトル、説明、カテゴリ、対象者、販売状態、価格、提供方法、CTAを見せる方針にした。
- 詳細ページで、対象者、販売状態、価格、提供方法、悩み、機能、提供内容、使い方、注意事項を表示する。
- 空項目は不自然に表示しない。
- `internal` / `internal_only` はMarket側に出さない。

## v0.8 自社フォーム化

- Googleフォーム連携は使わない。
- `request.html` を自社フォームUIとして整理した。
- `data/site-config.json` の `contact.mode` で送信モードを管理する。
- `demo`、`mailto`、`endpoint` の3モード設計にした。
- demo modeでは実送信せず、入力確認と準備中表示のみを行う。
- mailto modeでは公開用メールアドレス宛にメール作成する。
- endpoint modeでは将来自社APIへPOSTする設計にした。

## GitHub Pages上の制約

GitHub Pages単体ではサーバー側の送信処理ができない。

そのため、現時点では問い合わせ内容を保存せず、demo modeで確認表示だけを行う。
本送信を行う場合は、mailto modeまたは将来のendpoint modeへ切り替える。

## thanks.html の扱い

`thanks.html` はローカル検証用・導線確認用の完了画面。

自社フォームが未接続の場合、偽の送信完了として扱わない。

## まだ実装しないこと

- Supabase接続
- 認証
- 決済
- 自動納品
- 購入履歴管理
- 投稿者ログイン
- 管理画面本番化

## 禁止事項

- 問い合わせ内容をlocalStorageに保存しない。
- APIキーや秘密キーをフロントに置かない。
- 有料商品の本体ファイルをGitHubに置かない。
- 個人情報・顧客情報・勤怠実データを追加しない。
- `archive/` やzipを戻さない。
