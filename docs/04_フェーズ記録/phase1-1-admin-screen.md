## 推奨方針

現時点の推奨は、案A「方針整理のみ」。

次に進むなら、案B「mock管理者画面」で項目と導線を確認する。

本物の企業横断管理画面である案Dは、v1.1では非推奨。実装する場合は、管理者権限、RLS、API、監査、公開範囲について人間判断が必要。

## DB / RLS変更の要否

### 現在のRLSで企業横断一覧を取得できるか

通常ログインとanon/public keyの範囲では、自社データのみ取得できる前提。

企業横断一覧が取得できないことは、RLS上は正常な挙動。

### 管理者だけ横断取得するには必要になりそうなもの

- 管理者ユーザーの判定方法。
- 管理者フラグまたは管理者テーブル。
- RLS policyの追加または見直し。
- Supabase Edge Functionsなどの管理者専用API。
- GitHub Pages上のフロントから直接service_roleを使わない設計。
- 監査ログまたは操作ログの検討。

### v1.1でそこまでやるべきか

現時点ではやらない。

v1.1ではDB / RLS / migration変更を伴う本物の管理者画面には入らず、方針整理またはmock管理者画面までに留める。
