# phase1-3-purchase-start-flow-realignment

## Phase

v1.3 購入前後の利用開始フロー整理

## 状態

方針整理。

旧 `v1.3 アプリ追加申請` は、購入者が既存アプリの利用申請を出す導線として検討していたが、本線仕様から外す。

購入者側の「アプリ追加申請」は不要。

購入者は購入画面で「購入」または「利用開始」を押すと、そのアプリを使える設計にする。

## 決定事項

### 購入者側のアプリ追加申請

購入者側の「アプリ追加申請」は不要。

購入者向けに `app_add_requests` を使う導線は本線から外す。

購入者向けの基本フロー:

```text
購入画面で「購入」または「利用開始」を押す
↓
アプリが使える状態になる
↓
account.html の利用中アプリ / 新ポータル上で利用可能として表示される
↓
「開く」から別タブで起動できる
↓
必要に応じてサポートを申し込める
```

### アプリ状態ごとのCTA

| 状態 | CTA | 方針 |
|---|---|---|
| 無料アプリ | 利用開始 | 押すと利用可能状態にする。 |
| 有料アプリ | 購入する | 決済実装後は購入後に利用可能状態にする。 |
| 有料化予定アプリ | 利用開始 / 有料化予定 | 決済未実装中は無料アプリと同じように利用可能にする。 |
| β版アプリ | β版を試す | 押すと利用可能状態にする。 |
| 相談型アプリ | 通常カードには表示しない | 問い合わせ、導入相談、カスタマイズ相談、開発相談の導線に分離する。 |
| 準備中アプリ | 準備中 | 利用開始できない。 |
| 利用中アプリ | 開く | 別タブで起動する。 |

### 利用開始後の挙動

- 無料アプリの利用開始後: 別タブで開く。
- β版アプリの利用開始後: 別タブで開く。
- 有料アプリ購入後: 別タブで開く。

将来的には、どのアプリでも利用可能状態になったら以下の状態にする。

- `account.html` の利用中アプリに追加される。
- 新ポータル上で購入済み・利用可能として表示される。
- 「開く」から別タブで起動できる。

## 有料アプリの扱い

決済未実装中の有料予定アプリは、「有料化予定」と表示しつつ、当面は無料アプリと同じように利用可能にする。

決済実装後の想定:

```text
購入する
↓
決済
↓
app_instances に追加
↓
利用開始
```

将来的に管理者画面から変更できるようにしたい項目:

- 価格
- 無料 / 有料 / β版 / 準備中などの提供状態
- 有料化予定表示
- 公開 / 非公開

## 利用解除

利用解除ボタンを追加する方針。

ただし、誤操作防止のため複数回確認を入れる。

内部処理方針:

- 利用解除は `app_instances` の物理削除ではなく、`inactive` / `disabled` のような利用停止状態にする。
- 再利用開始で `active` に戻せるようにする。
- 履歴と復旧可能性を残す。

## 任意サポート

購入者は、アプリ利用申請ではなく、必要に応じてサポートを申し込む。

サポート種別:

- 初期設定サポート
- 使い方説明
- カスタマイズ相談
- 導入代行

将来検討:

- 月額サポートコース
- 月に一定回数まで相談可能なプラン

価格帯は後続フェーズで検討する。

- 購入者向けにはそもそも不要な導線であるため。

- drop、rename、用途変更は人間判断が必要。

## クリエイター / 開発者向けアプリ投稿

クリエイター / 開発者がアプリを追加・投稿する機能は、購入者導線とは別物として将来フェーズに分離する。

将来の流れ:

```text
クリエイター / 開発者がアプリを投稿する
↓
運営が審査する
↓
Works Marketに掲載する
↓
購入者が購入・利用開始できる
```

初期方式:

- 最初はログイン不要の投稿希望フォーム、または問い合わせフォームで受付する。
- 運営が手動確認する。
- クリエイターによる直接公開、DB自動登録、コード直接アップロードは後回しにする。

理由:

- クリエイターアカウント、権限管理、DB自動登録、コードアップロード、セキュリティ審査、売上分配は事故リスクが高い。
- 最初から自由投稿・直接公開にすると、リーガル、セキュリティ、運用事故が起きやすい。

将来検討項目:

- クリエイター登録
- アプリ投稿フォーム
- アプリ名
- 説明文
- 対象業種
- 価格
- デモURL
- サムネイル
- HTML / JS / CSSアップロード
- GitHub連携
- 審査ステータス
- 公開 / 非公開
- 売上分配
- 利用規約
- セキュリティ審査

初期実装で後回しにするもの:

- HTML / JS / CSS直接アップロード
- DB自動登録
- 直接公開
- 売上分配自動化
- クリエイター専用管理画面
