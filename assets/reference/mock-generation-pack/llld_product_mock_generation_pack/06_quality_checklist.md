# 06 Quality Checklist

## 商品カードモック品質チェック

### 一目でわかるか
- [ ] 何のアプリか3秒以内にわかる
- [ ] サムネイルが抽象的ではない
- [ ] アプリの主機能が見える
- [ ] タイトルが読みやすい
- [ ] カテゴリが読みやすい
- [ ] 説明文が短い

### カード構造
- [ ] 状態タグが適切
- [ ] サムネイルが上部に大きくある
- [ ] アイコンがある
- [ ] タイトルがある
- [ ] カテゴリバッジがある
- [ ] 説明文がある
- [ ] 「開く」ボタンがある
- [ ] 「お気に入り」ボタンがある
- [ ] 「詳細」ボタンがある
- [ ] その他メニューがある

### デザイン
- [ ] 白カード
- [ ] 角丸
- [ ] 薄い影
- [ ] 余白が広め
- [ ] ティールの主ボタン
- [ ] ダークネイビー系の文字
- [ ] 日本の業務アプリらしい
- [ ] LLLD Works Hubのカード群と並べても違和感がない

### サムネイル
- [ ] thumbnailTypeに合っている
- [ ] 本物のUIらしい
- [ ] 代表画面になっている
- [ ] 過度に細かくない
- [ ] 見せたい要素が中央または目立つ位置にある

### 日本語
- [ ] タイトルに誤字がない
- [ ] 説明文に誤字がない
- [ ] ボタン文言が正しい
- [ ] カテゴリ名が正しい
- [ ] 不自然な日本語がない

---

## HTML実装品質チェック

### 既存機能
- [ ] 検索が動く
- [ ] カテゴリ絞り込みが動く
- [ ] お気に入りが動く
- [ ] 利用履歴が動く
- [ ] 並び替えが動く
- [ ] 開くリンクが正しい
- [ ] 詳細ボタンが動く
- [ ] localStorageが壊れていない

### データ設計
- [ ] カードデータが配列またはJSONで管理されている
- [ ] idが一意
- [ ] thumbnailImageが使える
- [ ] thumbnailImageがない場合の代替がある
- [ ] thumbnailType不明時にgenericが出る
- [ ] カテゴリ追加に対応できる
- [ ] タグ追加に対応できる

### レスポンシブ
- [ ] PCで崩れない
- [ ] iPad幅で崩れない
- [ ] スマホ幅で崩れない
- [ ] ボタンが押しやすい
- [ ] サムネイルが潰れない

### GitHub Pages
- [ ] 相対パスで動く
- [ ] 外部API不要
- [ ] ビルド不要
- [ ] index.htmlで開ける
- [ ] コンソールエラーがない

---

## ビジコン見せ方チェック

- [ ] 単なるリンク集に見えない
- [ ] アプリが商品として見える
- [ ] 今後増えていくプラットフォーム感がある
- [ ] 「誰でもプログラマー」の世界観に合う
- [ ] 社内運用と外部販売の両方に発展できそう
- [ ] 現場発の小さなアプリが集約される印象がある
