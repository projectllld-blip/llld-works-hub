# 過去問PDF追加・分類ルール

入試問題印刷システムへ過去問PDFを追加するときの確認ルール。
ファイル名ではなく、PDF本文・表紙・欄外・解答解説ページの表記を優先する。

## 目的

- 学校、年度、入試区分、教科、資料種別を取り違えない。
- 印刷時に必要な問題用紙、解答用紙、解答、解説を探せる状態にする。
- 受信箱の元PDFは処理後に残さない。

## 基本の保存先

```text
apps/exam-print/data/
└── 都道府県_学校名_入試種別問題/
    └── 年度/
        └── 入試区分/
            └── 教科コード_教科名/
                └── 元号_入試区分_教科_資料種別.pdf
```

例:

```text
広島県_崇徳中学校_中学入試問題/2018/前期入試/01_国語/H30_前期入試_国語_問題・解答用紙.pdf
広島県_広島国際学院高等学校_高校入試問題/2019/一般入試/03_英語/H31_一般入試_英語_問題・解答用紙.pdf
```

## 教科コード

| コード | 教科 |
|---|---|
| 01 | 国語 |
| 02 | 算数・数学 |
| 03 | 英語 |
| 04 | 理科 |
| 05 | 社会 |
| 06 | 社会・理科など複合教科 |
| 90 | 配点表 |
| 99 | その他 |

## 資料種別

| 種別 | 判断基準 |
|---|---|
| 問題 | 問題本文のみ |
| 問題・解答用紙 | 問題本文と解答欄が同じPDFに入っている |
| 解答用紙 | 解答欄だけ |
| 解答 | 答えのみ |
| 解説 | 解説のみ |
| 解答・解説 | 解答集、解答例、解説が一体になっている |
| 配点表 | 配点だけ |
| その他 | リスニング原稿、表紙、注意書きなど |

## 必ず見る場所

1. 1ページ目の表題、学校名、年度、教科。
2. 各ページ欄外の年度表記。例: `2018(H30) 崇徳中`。
3. 解答用紙の見出し。例: `中学校前期入学試験問題 解答欄【算数】`。
4. 解答集の収録内容表。年度横断の解答集は、含まれる各年度へ同じPDFを置いてよい。
5. 最終ページ。解答用紙だけが末尾に付いていることがある。

## バラバラPDFへの対応

- 表紙なしで問題から始まるPDFは、欄外の年度・学校名・ページ番号で判断する。
- 同じ教科の問題が複数PDFに分割されている場合は、印刷事故を防ぐため連結して1PDFにする。
- 算数の問題PDFに算数と理科の解答用紙が入っているなど、複数教科の解答欄が混在する場合も、ページを分割または複製して必ず教科別フォルダへ置く。
- 複数教科が1冊に入っている問題冊子は、問題ページと解答用紙ページを教科別に切り出し、`問題・解答用紙` として登録する。
- 解答集が複数年度・複数教科を含む場合は、該当年度・該当教科の `解答・解説` として複製登録する。ページ内で複数教科が混在し切り抜きが危険な場合は、該当年度の解答解説ページを教科別PDFへ複製収録し、処理メモに残す。
- `全教科` という教科区分は原則使わない。どうしても教科特定できない資料だけ一時的に要確認へ回し、正式登録前に再確認する。

## セット化の原則

- 問題系は `問題・解答用紙` にそろえる。問題本文だけ、解答用紙だけに分かれている場合は、ページを結合して1PDFにする。
- 解答系は `解答・解説` にそろえる。解答例だけ、解説だけ、年度横断解答集の場合も、教科別PDFとして登録する。
- 問題本文または解答用紙が欠け、`問題・解答用紙` にできない場合は、無理に資料種別を偽らず `問題` または `解答用紙` として登録し、不足一覧へ書く。
- 中学入試などで英語が存在しない可能性が高い場合も、5教科チェックの観点では「英語資料なし」と明記する。

## 追加後の作業

1. `python3 generate_exam_data.py --data-dir data --out-dir .` を実行する。
2. `exam_manifest.json` と `exam_data.js` が更新されたことを確認する。
3. 一覧を生成し、学校・年度・教科・資料種別に不足がないか見る。
4. 受信箱の元ファイルを整理済みアーカイブへ移動し、受信箱を空にする。
5. GitHubリモートがある場合は、今回分だけをstageしてcommit/pushする。
